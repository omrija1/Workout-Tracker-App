
State Diagram for Use Case 6: Personalized Dashboard Display

States:
1. Initial State (Authenticated)
2. Fetching User Data
3. Fetching Upcoming Workouts
4. Fetching Recent Performance Metrics
5. Fetching Weight History
6. Displaying Dashboard
7. Navigating To Other Sections
8. Error State (Database fetch error)

Transitions and Conditions:
1. OpenApp(): Initial State -> Fetching User Data
   - Condition: User must be authenticated
2. FetchUserData() & ReturnUserData(): Fetching User Data -> Fetching Upcoming Workouts
3. FetchUpcomingWorkouts() & ReturnUpcomingWorkouts(): Fetching Upcoming Workouts -> Fetching Recent Performance Metrics
4. FetchRecentPerformanceMetrics() & ReturnRecentPerformanceMetrics(): Fetching Recent Performance Metrics -> Fetching Weight History
5. FetchWeightHistory() & ReturnWeightHistory(): Fetching Weight History -> Displaying Dashboard
6. NavigateToOtherSections(): Displaying Dashboard -> Navigating To Other Sections

Actions (Optional):
- Display an error message: If Database fails to fetch any data
