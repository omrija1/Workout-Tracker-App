
State Diagram for Use Case 3: Advanced Performance and Weight Tracking

States:
1. Initial State
2. Navigating to Performance Tracking
3. Fetching User Data
4. Displaying Performance Metrics
5. Updating Weight
6. Updating Weight in DB
7. Displaying Confirmation
8. Recalculating Performance Metrics
9. Displaying Updated Metrics
10. Error State (DB error or Update error)

Transitions and Conditions:
1. OpenPerformanceTracking(): Initial State -> Navigating to Performance Tracking
2. FetchUserData(): Navigating to Performance Tracking -> Fetching User Data
3. ReturnUserData(): Fetching User Data -> Displaying Performance Metrics
   - Condition: If error, go to Error State
4. UpdateWeight(new_weight): Displaying Performance Metrics -> Updating Weight
5. UpdateWeightInDB(new_weight): Updating Weight -> Updating Weight in DB
6. ConfirmUpdate(): Updating Weight in DB -> Displaying Confirmation
   - Condition: If error, go to Error State
7. RecalculatePerformanceMetrics(): Displaying Confirmation -> Recalculating Performance Metrics
8. ReturnUpdatedMetrics(): Recalculating Performance Metrics -> Displaying Updated Metrics

Actions (Optional):
- DisplayError(): If there is a database error
- DisplayUpdateError(): If there is an error in updating weight
