
State Diagram for Use Case 4: Dynamic Reminders and Notifications

States:
1. Initial State
2. Authenticated State
3. Navigating to Reminder Screen
4. Displaying Reminders
5. Adding/Editing Reminder
6. Filling Reminder Form
7. Saving Reminder
8. Scheduling Notification
9. Receiving Notification
10. Acknowledging Notification
11. Logging Notification
12. Deleting Reminder
13. Confirming Deletion
14. Error State (Validation or Deletion error)

Transitions and Conditions:
1. OpenApp(): Initial State -> Authenticated State
   - Condition: If not authenticated, redirect to login
2. NavigateToReminderScreen(): Authenticated State -> Navigating to Reminder Screen
3. FetchReminders() & ReturnReminders(): Navigating to Reminder Screen -> Displaying Reminders
4. AddOrEditReminder(): Displaying Reminders -> Adding/Editing Reminder
5. DisplayReminderForm(): Adding/Editing Reminder -> Filling Reminder Form
6. SubmitReminderForm(details): Filling Reminder Form -> Saving Reminder
   - Condition: If validation fails, return to Filling Reminder Form
7. SaveResult(): Saving Reminder -> Scheduling Notification
8. ScheduleNotification(reminderDetails) & NotificationStatus(): Scheduling Notification -> Displaying Reminders
9. PushNotification(): Scheduling Notification -> Receiving Notification
10. AcknowledgeNotification(): Receiving Notification -> Acknowledging Notification
11. UpdateNotificationStatus() & LogNotificationEvent() & LogStatus(): Acknowledging Notification -> Displaying Reminders
12. DeleteReminder(): Displaying Reminders -> Deleting Reminder
13. ConfirmDelete() & ConfirmAction(): Deleting Reminder -> Confirming Deletion
14. DeleteResult(): Confirming Deletion -> Displaying Reminders
   - Condition: If deletion fails, go to Error State

Actions (Optional):
- UpdateReminderScreen(): If a reminder is saved or deleted
- Display an error message: If validation or deletion fails
