
Sequence Diagram for Use Case 1: User Onboarding and System Initialization

1. User -> System: OpenApp()
  - Condition: If the user is not registered, proceed to step 2. Otherwise, go to step 12.

2. System -> MotivationalQuoteAPI: FetchDailyQuote()

3. MotivationalQuoteAPI -> System: ReturnQuote()

4. System -> User: DisplayWelcomeScreen(quote)

5. User -> System: SelectRegisterOrLogin()
  - Multiple Choices: Either Register or Login.

6. System -> User: DisplayForm(selectedOption)

7. User -> System: SubmitForm(details)

8. System -> Database: ValidateUser(details)
  - Condition: If validation fails, return to step 6.
  
9. Database -> System: ValidationResult()
  
10. System -> User: ShowValidationResult()
  
11. System -> User: RedirectToDashboard()

12. System -> User: DisplayDashboard()

Error Handling Flows:
- If FetchDailyQuote() fails, a default quote is displayed.
- If ValidateUser(details) fails, a validation error message is shown to the user.
